## 6) Como planejar URLs de logo e imagens (orientação objetiva para o editor)

### 6.1 Padrão recomendado para nomes de arquivos (organização)

Sugestão de estrutura (independente de onde vai hospedar)

* /assets/brand/logo-negativa.png

* /assets/pastores/denilson-mychelle.jpg
* /assets/pastores/jose-lourdes.jpg
* /assets/pastores/dyogo-dafne.jpg
* /assets/pastores/thiago-michelly.jpg
* /assets/banners/hero-1.jpg

### 6.2 Troca simples de URLs

O editor deve:

1. Colocar todas as URLs no registro único “Configurações do Site”.
2. Em cada página, puxar a URL via configuração (não “colar” a URL direto no bloco).
3. Para pastores, puxar foto e nomes via coleção “Pastores”.

Resultado

* Trocar o logo vira alterar 1 campo: site_logo_url.
* Trocar foto de um casal vira alterar 1 campo: pastor_foto_url.
* Atualizar banner hero vira alterar 1 campo: site_hero_banner_url.

### 6.3 Onde hospedar as imagens (opções)

Opção A (mínimo viável)

* Hospedar imagens no próprio Lovable (upload) e registrar as URLs na coleção “Configurações do Site”.

Opção B (mais profissional)

* Hospedar em um CDN (ex.: Cloudflare / DigitalOcean Spaces / outro) e usar URLs definitivas.
* Benefício: performance e controle.
